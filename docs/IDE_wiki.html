SystemScript IDE

A BETA development environment for the SystemScript programming language

SystemScript IDE is a BETA "Integrated Development Environment" specifically designed for SystemScript language development. It combines code editing capabilities with robust project management tools, making it ideal for everything from small scripts to complex system-level applications.

Key Features

  • Modern Editor: Advanced syntax highlighting, code completion, and error detection
  • Project Management: Organize files with intuitive project structures and templates
  • Integrated Build Tools: Compile and run SystemScript code directly from the IDE
  • Debugging: Step through code, inspect variables, and analyze execution flow
  • Customizable Interface: Dark mode, themes, and layout options to suit your workflow
  • Plugin System: Extend functionality with community-created plugins

Version Information: This documentation covers SystemScript IDE version 2.0.0. Features may vary in different versions.

SystemScript IDE is available for Windows, macOS, and Linux. Choose the appropriate installation method for your platform.

Code Editor

The code editor is the core component of SystemScript IDE, providing advanced editing capabilities specifically tailored for SystemScript development.

Syntax Highlighting

The editor automatically highlights SystemScript syntax elements to improve code readability:

  • Keywords: module, import, fn, struct, etc.
  • Types: i32, f64, bool, str, etc.
  • Functions: println, calculate, etc.
  • Strings: "Hello, World!"
  • Comments: // This is a comment
  • Numbers: 42, 3.14
Syntax Highlighting Example
// SystemScript example
module main;

import system.io;

fn calculate(x: i32, y: i32) -> i32 {
    const result: i32 = x * y + 42;
    if (result > 100) {
        io.println("Large result: " + result);
    }
    return result;
}

fn main() -> i32 {
    const a: i32 = 10;
    const b: i32 = 20;
    return calculate(a, b);
}

Code Completion (AI code completion has not been implemented)

The IDE provides intelligent code completion to speed up your coding workflow:

  • Automatic Suggestions: As you type, the editor shows relevant completion options
  • Context-Aware Completion: Suggestions are based on current code context
  • Module Completion: Automatically suggests available module imports
  • Function Parameter Hints: Displays parameter types and names

Trigger code completion manually with Ctrl+Space or configure it to appear automatically after you type specific characters.

Code Folding

Code folding allows you to collapse blocks of code to focus on specific sections:

  • Click the - symbol in the gutter to collapse a code block
  • Click the + symbol to expand a collapsed block
  • Use Ctrl+Shift+[ to collapse and Ctrl+Shift+] to expand
  • Right-click in the gutter for additional folding options

Search & Replace

The IDE offers powerful search and replace functionality:

Single File Search

  • Press Ctrl+F to open the search dialog
  • Press Ctrl+H for search and replace
  • Options include case sensitivity, whole word matching, and regular expressions
  • Use F3 to find next and Shift+F3 to find previous

Project-Wide Search

  • Press Ctrl+Shift+F to search across all project files
  • Results appear in the Output panel with links to matching locations
  • Filter search by file types or specific directories
  • Replace across multiple files with Ctrl+Shift+H

Build & Run

SystemScript IDE integrates tightly with the SystemScript compiler, allowing you to build and run your code directly from the IDE.

Compiler Settings

Configure the compiler through the Settings dialog:

  1. Select Tools â†’ Settings from the menu bar
  2. Navigate to the Compiler tab
  3. Configure compiler path, build flags, and other options

Built-in vs External Compiler

SystemScript IDE offers two compiler options:

  • Built-in Compiler: Included with the IDE, provides basic functionality without external dependencies
  • External Compiler: More powerful, supports advanced optimizations and features

For production use or advanced features, we recommend configuring an external compiler. The built-in compiler is suitable for learning and simple projects.

Common Build Flags

Flag Description
--optimize Enable performance optimizations
--debug Include debug information in the output
--warnings Show additional warnings during compilation
--werror Treat warnings as errors
--strict Enable strict type checking

Building Projects

There are several ways to build your SystemScript project:

Building a Single File

  1. Open the file in the editor
  2. Press F7 or click the Build button in the toolbar
  3. View build results in the Output panel

Building an Entire Project

  1. Press Ctrl+F7 or select Build â†’ Build Project from the menu
  2. The IDE will build the main file and all dependencies
  3. Output files are placed in the project's build directory

Note: SystemScript IDE automatically saves all modified files before building to ensure the latest changes are compiled.

Running Projects

After a successful build, you can run your SystemScript application:

Running the Current File

  1. Press F5 or click the Run button in the toolbar
  2. If the file hasn't been built yet, the IDE will build it first
  3. Program output appears in the Output panel

Run Configuration Options

Configure how your application runs through Build â†’ Run Configuration:

  • Set command-line arguments
  • Configure environment variables
  • Specify working directory
  • Set up external terminal options

Debugging

SystemScript IDE includes a powerful debugger for analyzing and fixing issues:

Starting a Debug Session

  1. Set breakpoints by clicking in the gutter next to line numbers
  2. Press F9 or select Build â†’ Debug from the menu
  3. The program will run until it hits a breakpoint

Debugging Controls

  • F9: Start/continue debugging
  • F8: Step over (execute current line without stepping into functions)
  • F7: Step into (follow execution into functions)
  • Shift+F8: Step out (continue until current function returns)
  • Shift+F5: Stop debugging

Debug Views

While debugging, additional panels become available:

  • Variables: Shows current variable values
  • Call Stack: Displays the execution call hierarchy
  • Breakpoints: Lists and manages all breakpoints
  • Watch: Monitors custom expressions during execution

Common Issues

This section covers solutions for common issues you might encounter while using SystemScript IDE.

Startup Issues

IDE Won't Start

  1. Check Prerequisites: Ensure your system meets the minimum requirements
  2. Verify Installation: Reinstall if necessary
  3. Check Logs: Examine logs in ~/.systemscript-ide/logs/
  4. Reset Configuration: Rename or delete ~/.systemscript-ide/config.json

Editor Issues

Syntax Highlighting Not Working

  1. Verify File Type: Ensure the file has the correct .ss extension
  2. Check Theme: Try switching to the default theme in Settings
  3. Reset Editor Settings: Go to Tools â†’ Settings â†’ Editor and reset to defaults

Code Completion Not Working

  1. Check If Enabled: Verify in Tools â†’ Settings â†’ Editor â†’ Auto-completion
  2. Trigger Manually: Try pressing Ctrl+Space
  3. Project Structure: Ensure proper imports and project structure
  4. Reset Cache: Select Tools â†’ Reset Caches

Build Issues

Compiler Not Found

  1. Check Compiler Path: Verify in Tools â†’ Settings â†’ Compiler
  2. Use Built-in Compiler: Enable the built-in compiler option
  3. Environment Variables: Check if compiler directory is in your PATH

Build Fails with No Error Message

  1. Check Output Panel: Look for errors in the Build tab
  2. Examine Build Flags: Some flags may cause silent failures
  3. File Permissions: Ensure write permissions for the build directory
  4. External Dependencies: Verify all required libraries are available

Performance Issues

IDE Running Slowly

  1. Close Unused Projects: Multiple open projects consume resources
  2. Disable Plugins: Temporarily disable non-essential plugins
  3. Increase Memory Allocation: Edit systemscript-ide.ini to allocate more memory
  4. Clear Caches: Select Tools â†’ Reset Caches and Restart

Error Codes

SystemScript IDE may display error codes during operation. This section explains common error codes and their solutions.

IDE Error Codes

Error Code Description Solution
IDE001 Configuration file corrupted Delete or rename ~/.systemscript-ide/config.json and restart
IDE002 Plugin load failure Disable problematic plugin in safe mode or delete from plugins directory
IDE003 Project file format error Ensure project.json is properly formatted
IDE004 Insufficient permissions Run the IDE with appropriate permissions or fix file/directory permissions
IDE005 Memory allocation error Increase available memory or reduce project size

Compiler Error Codes

Error Code Description Solution
SSC001 Syntax error Check syntax at indicated location
SSC002 Undefined symbol Ensure all variables and functions are properly declared
SSC003 Type mismatch Verify type compatibility in operations
SSC004 Module not found Check import paths and module availability
SSC005 Memory safety violation Fix unsafe memory access or use appropriate safety annotations

Runtime Error Codes

Error Code Description Solution
SSR001 Null pointer dereference Add null checks before accessing pointers
SSR002 Out of bounds access Ensure array indices are within valid range
SSR003 Stack overflow Check for infinite recursion or excessive local variables
SSR004 Heap memory exhaustion Fix memory leaks or increase available memory
SSR005 Division by zero Add validation before division operations

Performance Tips

Optimize your SystemScript IDE experience with these performance tips.

IDE Optimization

Improving IDE Performance

  • Adjust Memory Allocation

    Increase memory allocation in systemscript-ide.ini by setting -Xmx to a higher value, such as -Xmx2048m for 2GB.

  • Disable Unused Plugins

    Disable plugins you don't regularly use to reduce startup time and resource usage.

  • Optimize Project Size

    Break very large projects into smaller modules. Mark source roots correctly to improve indexing.

  • Configure Auto-Save

    Adjust auto-save frequency to balance between data safety and performance.

  • Use Local History Judiciously

    Configure local history to keep fewer revisions if disk space is limited.

Editor Optimization

Speeding Up the Editor

  • Disable Real-time Code Analysis

    For very large files, consider disabling real-time code analysis in Tools â†’ Settings â†’ Editor â†’ Code Analysis.

  • Limit Syntax Highlighting

    For extremely large files, you can disable syntax highlighting by right-clicking the editor tab and selecting Disable Syntax Highlighting.

  • Optimize Font Rendering

    Some fonts perform better than others. JetBrains Mono and Source Code Pro are optimized for IDE use.

  • Disable Minimap for Large Files

    The minimap can consume resources for very large files. Disable it in Tools â†’ Settings â†’ Editor.

Building and Running

Optimizing Build and Run Performance

  • Use Incremental Builds

    Enable incremental builds in project settings to rebuild only modified files.

  • Configure Output Filtering

    Filter unnecessary output in the build console to improve performance during builds with verbose output.

  • Use Build Cache

    Enable compiler cache in Tools â†’ Settings â†’ Compiler â†’ Advanced to speed up repeated builds.

  • Optimize Run Configurations

    Configure run configurations to use appropriate memory settings for your applications.

Hardware Recommendations

For optimal performance with large projects, consider these hardware recommendations:

  • CPU: Modern multi-core processor (4+ cores recommended)
  • RAM: 16GB or more for large projects
  • Storage: SSD with at least 500MB/s read/write speeds
  • Graphics: Dedicated GPU can help with UI rendering
